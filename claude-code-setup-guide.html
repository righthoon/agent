STEP00

시작 전 준비물 확인

설치가 막히는 대부분의 이유는 준비물이 빠져서예요. 먼저 여기부터.

세 가지만 있으면 됩니다. 하나씩 확인하고 체크하세요.

1Node.js 18 이상

Claude Code를 npm으로 설치하려면 Node.js가 필요합니다. 터미널을 열고(맥: ⌘+Space → "터미널" / 윈도우: ⊞ → "PowerShell") 아래를 붙여넣어 버전을 확인하세요.

terminal · 버전 확인
node -v   # v18.0.0 이상이면 OK
npm -v
ℹ️ "command not found: node" 가 뜨면?
Node.js가 없는 겁니다. nodejs.org 에서 LTS 버전을 내려받아 설치(그냥 다음→다음)한 뒤, 터미널을 껐다 다시 열고 위 명령을 다시 실행하세요.

2Claude 계정 (Pro 또는 Max 권장)

claude.ai 계정이 필요합니다. 해커톤 동안 스킬을 원활히 쓰려면 Pro 이상 구독을 권장합니다. 로그인은 설치 후 브라우저로 진행합니다.

3(선택) GitHub 계정

/ultraplan 클라우드 기능을 쓰려면 GitHub 계정과 저장소가 필요합니다. 지금 없어도 나머지는 전부 진행 가능하니, 없으면 넘어가도 됩니다.

STEP01

Claude Code 설치 & 첫 실행

터미널에서 대화하는 AI 에이전트, "하네스"를 깔아봅니다.

A설치 (macOS · Windows 공통)

가장 확실한 방법은 npm 설치입니다. 아래 한 줄을 복사해 붙여넣고 Enter.

terminal · 공통
npm install -g @anthropic-ai/claude-code
🍎 macOS 대안 (Node 없이 설치)
Node 설치가 번거롭다면 맥/리눅스는 네이티브 설치 스크립트도 가능합니다.
terminal · macOS/Linux
curl -fsSL https://claude.com/install.sh | bash
⚠️ 윈도우에서 권한/EACCES 에러가 나면
PowerShell을 관리자 권한으로 실행한 뒤 다시 시도하세요. macOS에서 EACCES 권한 에러가 나면 sudo npm install -g @anthropic-ai/claude-code 로 설치합니다.

B첫 실행 & 로그인

작업할 폴더로 이동한 뒤 claude 를 실행합니다. 이번 해커톤 폴더를 예로 들면:

terminal
# 원하는 작업 폴더로 이동 (예시)
cd ~/hackathon

# Claude Code 실행
claude

처음 실행하면 브라우저가 열리며 로그인 화면이 뜹니다. Claude 계정으로 로그인하고 터미널로 돌아오면 준비 끝. (이미 실행 중이라면 /login 으로도 로그인할 수 있어요.)

C버전 확인 — 중요

/ultraplan 같은 최신 기능은 v2.1.91 이상에서 동작합니다. 버전을 확인하세요.

terminal
claude --version
ℹ️ 버전이 낮으면
npm install -g @anthropic-ai/claude-code@latest 로 최신 버전으로 업데이트하세요.
STEP02

하네스 기본 설정 익히기

플러그인을 깔기 전에, 자주 쓰는 기본 명령 4개만 알고 갑시다.

Claude Code가 실행된 상태(프롬프트에 > 가 보이는 상태)에서 아래 슬래시 명령들을 입력해 보세요. 슬래시(/)를 치면 자동완성 목록이 뜹니다.

명령하는 일
/help사용 가능한 모든 명령·스킬 목록 보기 (막히면 제일 먼저)
/model모델 선택 — 어려운 작업엔 Opus, 가벼운 작업엔 Sonnet
/config테마·권한 모드 등 환경 설정
/clear대화 맥락 초기화 (새 작업 시작할 때)
💡 CLAUDE.md 개념 (한 줄 요약)
작업 폴더에 CLAUDE.md 파일을 두면 그 안에 적은 규칙을 Claude가 항상 참고합니다. "코딩 스타일, 지켜야 할 규칙"을 여기 적어두면 매번 설명할 필요가 없어요. 지금은 몰라도 됩니다 — 나중에 /init 으로 자동 생성할 수 있어요.
⚠️ 폴더 신뢰 프롬프트
처음 폴더에서 실행하면 "이 폴더를 신뢰하겠냐"고 물어봅니다. 본인이 만든/받은 폴더일 때만 신뢰하세요.
STEP03

내장 스킬 — 설치 없이 바로 쓰기

deep-research 와 ultraplan 은 Claude Code에 이미 들어 있어요. 확인만 하면 됩니다.

A/deep-research — 심층 리서치

여러 웹 출처를 검색·교차검증해서 인용이 붙은 리포트를 만들어 줍니다. 시장 조사, 논문 배경 조사, 경쟁 분석에 유용합니다. 그냥 이렇게 쓰면 됩니다:

claude 프롬프트
/deep-research 국내 원격의료 규제 현황과 최근 2년 변화
ℹ️ 질문은 구체적으로
범위가 넓으면 되묻습니다. "예산/대상/지역" 같은 조건을 함께 주면 결과가 훨씬 좋아집니다.

B/ultraplan — 클라우드 계획 (프리뷰)

복잡한 작업의 실행 계획을 클라우드에서 대신 세워주는 내장 기능입니다. 터미널은 계속 다른 일에 쓰고, 계획이 완성되면 브라우저에서 문단별로 코멘트하며 다듬을 수 있어요.

claude 프롬프트
/ultraplan 환자 문진 데이터를 요약하는 웹앱 만들기
⚠️ ultraplan 사용 조건
research preview 기능으로 Claude Code v2.1.91 이상, Claude Code on the web 계정, GitHub 저장소가 필요합니다. (Bedrock/Vertex/Foundry 환경에서는 미지원.) 조건이 안 되면 로컬 plan mode로 계획을 세워도 충분합니다.
STEP04

플러그인 설치, 이 패턴이 전부예요

앞으로 4개를 깔 텐데 방법은 똑같습니다. 딱 두 단계.

모든 플러그인은 ①마켓플레이스 등록 → ②설치 순서로 진행합니다. Claude Code가 실행된 상태에서 슬래시 명령으로 입력하세요.

  1. /plugin marketplace add <주소> — 플러그인이 있는 "상점"을 등록
  2. /plugin install <이름>@<상점> — 그 상점에서 플러그인 설치
💡 /plugin 만 쳐도 됩니다
명령을 외우기 부담되면 그냥 /plugin 을 실행하세요. 마켓플레이스 추가·설치·삭제를 메뉴(화살표 키로 선택)로 할 수 있는 UI가 열립니다.
⚠️ 설치 후엔 재시작
플러그인은 Claude Code를 껐다 켜야(exit 후 claude 재실행) 완전히 적용됩니다. 4개 다 깔고 마지막에 한 번 재시작해도 됩니다.
STEP05

superpowers 설치

브레인스토밍·TDD·체계적 디버깅 등 "일 잘하는 방법" 워크플로우 모음.

Claude Code 안에서 아래 두 줄을 한 줄씩 실행하세요.

claude 프롬프트 · ① 마켓 등록
/plugin marketplace add obra/superpowers-marketplace
claude 프롬프트 · ② 설치
/plugin install superpowers@superpowers-marketplace
💡 확인 방법
재시작 후 /help 목록에 superpowers:brainstorming, superpowers:test-driven-development 등이 보이면 성공입니다. 뭔가 새로 만들 땐 brainstorming 이 자동으로 먼저 도와줍니다.
STEP06

oh-my-claudecode 설치 (→ ultrawork)

여러 에이전트를 동시에 굴리는 병렬 실행 엔진 "ultrawork" 를 제공합니다.

마찬가지로 두 줄. 이 플러그인은 마켓 주소로 GitHub URL을 직접 씁니다.

claude 프롬프트 · ① 마켓 등록
/plugin marketplace add https://github.com/Yeachan-Heo/oh-my-claudecode
claude 프롬프트 · ② 설치
/plugin install oh-my-claudecode

쓰는 법ultrawork

병렬로 빠르게 처리하고 싶은 작업은 /ultrawork 로 시작하거나, 평소 프롬프트 맨 앞에 ulw 를 붙이면 됩니다.

claude 프롬프트
/ultrawork 랜딩 페이지 3개 섹션을 각각 만들어 합쳐줘
ℹ️ tmux (선택)
/team 같은 고급 멀티 에이전트 기능은 tmux 가 필요합니다. 해커톤 기본 작업엔 없어도 됩니다. 맥은 brew install tmux 로 설치 가능.
STEP07

andrej-karpathy-skills 설치

"단순하게, 필요한 것만, 외과적으로" — 잘못된 가정과 과도한 변경을 막는 코딩 원칙.

방법 A플러그인으로 설치 (권장)

모든 프로젝트에서 재사용되는 방식입니다.

claude 프롬프트 · ① 마켓 등록
/plugin marketplace add forrestchang/andrej-karpathy-skills
claude 프롬프트 · ② 설치
/plugin install andrej-karpathy-skills@karpathy-skills

방법 B특정 프로젝트에만 (CLAUDE.md 방식)

플러그인 대신, 원칙을 프로젝트의 CLAUDE.md 에 직접 넣고 싶다면 터미널(claude 밖)에서 프로젝트 폴더로 이동해 실행하세요.

terminal · 새 프로젝트
curl -o CLAUDE.md https://raw.githubusercontent.com/forrestchang/andrej-karpathy-skills/main/CLAUDE.md
⚠️ 기존 CLAUDE.md가 있으면
위 명령은 덮어씁니다. 이미 파일이 있다면 덮어쓰지 말고 이어붙이기를 쓰세요:
terminal · 기존 프로젝트
curl https://raw.githubusercontent.com/forrestchang/andrej-karpathy-skills/main/CLAUDE.md >> CLAUDE.md
STEP08

humanizer 설치

위키피디아 "Signs of AI Writing" 기반으로 AI 글쓰기 33개 패턴을 찾아내 자연스러운 글로 고쳐 줍니다.

발표 자료·제출 문서·이메일의 글에서 "AI가 쓴 티"를 걷어낼 때 유용합니다. blader/humanizer 저장소 기반. 설치는 동일한 두 줄.

claude 프롬프트 · ① 마켓 등록
/plugin marketplace add blader/humanizer
claude 프롬프트 · ② 설치
/plugin install humanizer@humanizer
ℹ️ 대안 설치 (skills CLI)
플러그인 대신 터미널에서 스킬만 바로 추가할 수도 있어요.
terminal
npx skills add blader/humanizer

무엇을이런 "AI 티"를 잡아냅니다

탐지하는 33개 패턴을 5가지 갈래로 정리하면:

갈래고쳐 주는 패턴 예시
내용의미 부풀리기, 유명세 이름 끼워넣기, 피상적인 "-ing" 분석, 홍보성 표현, 모호한 출처("전문가들은…"), 판에 박힌 "도전 과제" 나열
어휘AI 상투어(actually, additionally, testament, landscape, delve), be동사 회피(serves as, boasts), 부정 대구, 3개 나열 습관, 유의어 돌려쓰기, 수동태 남용
스타일대시(—) 남용, 볼드 남용, 제목 Title Case, 이모지, 표지판식 안내 문구, 억지 결정타(punchline), 격언 흉내, 수사적 질문으로 시작하기
대화 흔적챗봇 잔재("I hope this helps!"), 지식 컷오프 면책 문구, 아부성 어조
군더더기불필요한 필러("in order to", "due to the fact that"), 과도한 완충 표현(hedging), 뻔한 마무리 결론

쓰는 법붙여넣고 /humanizer

다듬고 싶은 글을 붙여넣고 함께 실행하면, 뜻과 어조는 유지한 채 자연스럽게 고쳐 줍니다.

claude 프롬프트
/humanizer
아래 발표 소개 문단을 자연스럽게 다듬어줘:
"본 프로젝트는 의료 현장의 도전 과제를 해결하는 혁신적인 여정입니다..."
💡 내 문체로 맞추고 싶다면
먼저 내가 쓴 글 샘플 몇 개를 보여준 뒤 다듬어 달라고 하면, 그 목소리(voice)에 맞춰 교정합니다.

FIX10

자주 막히는 곳 & 해결

에러가 떠도 당황 금지. 대부분 아래에 답이 있어요.
ERR 터미널에 claude: command not found 가 떠요▸

설치가 안 됐거나 PATH 문제입니다. ① node -v 로 Node가 있는지 확인 → 없으면 STEP 00 참고. ② npm install -g @anthropic-ai/claude-code 재실행. ③ 터미널을 완전히 껐다 다시 열기. 그래도 안 되면 맥은 네이티브 설치(curl … install.sh | bash)를 시도하세요.

ERR npm 설치 중 EACCES / 권한 오류▸

맥/리눅스: sudo npm install -g @anthropic-ai/claude-code. 윈도우: PowerShell을 관리자 권한으로 실행 후 재시도. (더 깔끔한 방법은 nvm으로 Node를 사용자 폴더에 설치하는 것입니다.)

ERR 플러그인을 깔았는데 /help 에 안 보여요▸

거의 항상 재시작으로 해결됩니다. exit 로 나갔다가 claude 로 다시 실행하세요. 그래도 없으면 /plugin 에서 해당 플러그인이 Enabled 인지, 마켓플레이스가 정상 등록됐는지 확인합니다.

ERR /ultraplan 이 동작하지 않아요▸

ultraplan은 프리뷰 기능이라 조건이 있습니다. ① claude --version 이 v2.1.91 이상인지 (아니면 @latest 업데이트). ② Claude Code on the web 계정 + GitHub 저장소가 있는지. 조건이 안 되면 그냥 로컬 plan mode로 계획을 세우면 됩니다.

ERR 마켓플레이스 add 가 실패해요▸

주소 오타를 먼저 확인하세요(특히 Yeachan-Heo 의 대소문자·하이픈). 사내망/방화벽 환경이면 GitHub 접근이 막혔을 수 있습니다. /plugin UI 메뉴로 등록을 다시 시도하고, 네트워크를 바꿔(핫스팟 등) 재시도해 보세요.

TIP 화면의 한글이 깨져 보여요▸

터미널 인코딩을 UTF-8로 설정하세요. 윈도우 기본 터미널보다 Windows Terminal 또는 VS Code 내장 터미널 사용을 권장합니다. VS Code 확장(Claude Code)을 쓰면 가장 편합니다.
